CORE CONCEPTS:

FRONTEND STATE MACHINE:
  Idle → Breathing → Prompt → Speaking → Feedback → Idle (restart)

BACKEND MESSAGE FLOW:
  Client: { topic: "session:user_session", event: "finished_speaking", ... }
  Server: { topic: "session:user_session", event: "phx_reply", payload: { response: { feedback: "..." } } }

COMMUNICATION LAYER:
  Elm ←→ Ports (JavaScript) ←→ WebSocket ←→ Phoenix Channels ←→ Elixir

==============================================
RUNNING THE POC:

Terminal 1: cd backend && mix phx.server
Terminal 2: cd frontend && python3 -m http.server 3000
Browser:   http://localhost:3000
